## What Are Distractors?

**Distractors** are **irrelevant context passages** mixed into the input alongside the relevant one. They simulate what happens in real RAG: the retriever often returns multiple documents, some relevant and some not.

```
WITHOUT distractors (current):
  instruction = [Relevant Pasal X]
  input       = "Question about Pasal X?"
  output      = "Answer from Pasal X"

WITH distractors (should add):
  instruction = [Irrelevant Pasal A] + [Relevant Pasal X] + [Irrelevant Pasal B]
  input       = "Question about Pasal X?"
  output      = "Answer from Pasal X ONLY (ignoring A and B)"
```

## Why This Matters for Your Use Case

In a real RAG system for peraturan desa, the retriever might return:

```
Retriever returns 5 passages for query "Berapa biaya air bersih di Desa Biru?":

  1. [RELEVANT]   Desa Biru Pasal 8: "Biaya pemasangan Rp. 300.000..."
  2. [DISTRACTOR] Desa Cigentur Pasal 8: "Tunjangan BPD ditetapkan..."
  3. [DISTRACTOR] Desa Sukapura Pasal 4: "Syarat calon Kepala Desa..."
  4. [RELEVANT]   Desa Biru Pasal 6: "Pengelolaan oleh BUMDesa..."
  5. [DISTRACTOR] Desa Nanjung Pasal 3: "Pendapatan desa bersumber dari..."
```

Without distractor training, the model might **mix information** from different desa or **hallucinate** answers.
